# 命令行参考

RedScript 命令行工具，用于将 `.mcrs` 文件编译为数据包。

## 安装

```bash
npm install -g redscript-mc
```

## 命令

### redscript compile

将 RedScript 文件编译为 Minecraft 数据包。

```bash
redscript compile <input> [options]
```

**参数：**

| 参数 | 描述 |
|------|------|
| `<input>` | `.mcrs` 文件或目录路径 |

**选项：**

| 选项 | 描述 | 默认值 |
|------|------|--------|
| `-o, --output <path>` | 输出目录 | `./out` |
| `-n, --namespace <name>` | 数据包命名空间 | 文件名 |
| `--pack-format <version>` | 包格式版本 | 最新版 |
| `--description <text>` | 包描述 | `"Generated by RedScript"` |
| `--minify` | 最小化输出 | `false` |
| `--no-comments` | 移除输出中的注释 | `false` |

**示例：**

```bash
# 基本编译
redscript compile hello.mcrs

# 指定输出目录
redscript compile hello.mcrs -o ./my-datapack

# 设置命名空间和描述
redscript compile game.mcrs -n minigame --description "My Mini Game"

# 最小化输出
redscript compile game.mcrs -o ./dist --minify
```

### redscript watch

监听文件变化并自动重新编译。

```bash
redscript watch <input> [options]
```

接受与 `compile` 相同的选项。

```bash
redscript watch game.mcrs -o ./server/world/datapacks/my-game
```

### redscript init

创建新的 RedScript 项目和样板文件。

```bash
redscript init [name]
```

```bash
redscript init my-datapack
```

创建：
```
my-datapack/
├── main.mcrs
└── redscript.json
```

### redscript check

对文件进行类型检查，不产生输出。

```bash
redscript check <input>
```

```bash
redscript check game.mcrs
```

### redscript version

显示版本信息。

```bash
redscript --version
```

## 配置文件

`redscript.json` — 可选的项目配置：

```json
{
  "namespace": "my_game",
  "description": "My Minecraft Mini Game",
  "packFormat": 48,
  "entry": "main.mcrs",
  "output": "./dist",
  "minify": false
}
```

| 字段 | 描述 |
|------|------|
| `namespace` | 数据包命名空间 |
| `description` | 包描述 |
| `packFormat` | Minecraft 包格式版本 |
| `entry` | 入口文件 |
| `output` | 输出目录 |
| `minify` | 最小化输出 |

## 退出码

| 退出码 | 含义 |
|--------|------|
| `0` | 成功 |
| `1` | 编译错误 |
| `2` | 文件未找到 |
| `3` | 配置无效 |
